Trabajo con R

Trabajo de Estadística con R

El objetivo de este trabajo es poner en práctica los conocimientos y destrezas adquiridas en el manejo de R - Cada grupo tendrá que realizar un plan de Marketing distinto para un producto o servicio en un contexto real de manera que las decisiones de ese plan estén basada en el análisis de datos con Python. - Los alumnos tendrán que buscar una base de datos o varias bases de datos con suficiente información para poder tomar decisiones de Marketing a partir de su análisis. Por otro lado, para garantizar que la parte de programación tuviese un mínimo de complejidad para poder poner en práctica la mayoría de lo aprendido durante el curso, se establecieron los siguientes requisitos para esas bases de datos: Un mínimo de 8 o 10 variables, que incluyesen tanto variables cualitativas como cuantitativas, un mínimo de 500 registros. Antes de empezar a trabajar con las bases de datos, estas deben ser validadas tanto por los profesores de Marketing como de Fundamentos de Computadores.
Dado que la obtención de datos en abierto no es fácil, se planteó a los alumnos varios portales con bases de datos abiertas para facilitar su búsqueda: - https://data.world/datasets/marketing - https://www.kaggle.com/datasets - https://datasetsearch.research.google.com/ - https://www.cmswire.com/digital-marketing/6-datasets-for-marketers-should-know-about/ - https://www.census.gov/data.html
- https://www.census.gov/acs/www/data/data-tables-and-tools/ - https://catalog.data.gov/dataset?res_format=CSV

Por otro lado, también se seleccionan varias bases de datos que satisfacen los requisitos anteriores para que los alumnos que no consigan encontrar una base de datos adecuada pudieran usarlas:

  • Datos sobre clientes de varias campañas de marketing: https://www.kaggle.com/rodsaldanha/arketing-campaign Este conjunto de datos se ha utilizado para un trabajo final de un curso en Business Analytics (https://www.kaggle.com/jackdaoud/marketing-data)

  • Datos sobre los clientes de un banco y las campañas de créditos: https://www.kaggle.com/henriqueyamahata/bank-marketing

  • Datos sociodemográficos sobre los clientes y su valor para una empresa (pude ser útil para diseñar estrategias para fidelizar clientes): https://www.kaggle.com/pankajjsh06/ibm-watson-marketing-customer-value-data

  • Datos de productos vendidos en la plataforma Wish: https://www.kaggle.com/jmmvutu/summer-products-and-sales-in-ecommerce-wish

  • Datos sobre abandono de clientes: https://www.kaggle.com/blastchar/telco-customer-churn

  • Datos sociodemográficos para la segmentación de clientes de una compañía de automóviles: https://www.kaggle.com/vetrirah/customer

  • Datos sobre campañas de publicidad en Facebook: https://www.kaggle.com/loveall/clicks-conversion-tracking

  • Datos AIRBNB de Madrid (estos son los datos que utilizaron en el trabajo del año pasado): https://www.kaggle.com/rusiano/madrid-airbnb-data

  • Una vez elegida la temática del plan de Marketing y la base de datos. Los grupos empezarían por definir los objetivos del plan de Marketing y concretar las preguntas que tenían que responder a partir del análisis de la base de datos.

  • A partir de esas preguntas, se definirán las tareas concretas de programación que deberían desarrollar.

  • La parte de programación se realizará en el entorno de desarrollo Replit. Esta plataforma permite la creación de proyectos de programación compartidos por grupos de manera que los alumnos de un mismo grupo pueden compartir el código. Al mismo tiempo permite la supervisión del profesor y la realización de comentarios por parte de este. Finalmente también dispone de control de versiones.

  • Cada grupo tendrá asignado como tutor uno de los profesores de Fundamentos de Computadores, y tendrán al menos una tutoría semanal para supervisar la marcha del trabajo.

  • La evaluación del trabajo será independiente para Marketing y Fundamentos de Computadores, ya en cada asignatura se evalúan desempeños distintos. De esta manera un alumno podría aprobar el trabajo en una de las asignaturas y suspenderlo en la otra.

  • La evaluación de la parte de programación en Fundamentos de computadores será grupal, es decir, se otorgará una nota al trabajo y no a cada alumno. Serán los integrantes del grupo quienes se distribuyan la nota del trabajo en función de sus méritos. Los alumnos remitirán a su tutor un documento consensuado con el reparto de la nota.

  • El trabajo se evaluará de acuerdo a la siguiente rúbrica:
    Mínimos

    • Definir funciones lógicas con el número de parámetros adecuado.
    • Crear un DataFrame a partir de un fichero csv o excel con el índice, separador de campos y separador de decimales adecuado.
    • Preprocesar el fichero para seleccionar las columnas necesarias y eliminar las filas con valores desconocidos.
    • Generar columnas nuevas a partir de la aplicación de fórmulas a las columnas existentes (tanto numéricas, como cadena y booleanas).
    • Aplicar filtros de al menos dos condiciones.
    • Calcular tablas de frecuencias simples y agrupadas según una o varias variables cualitativas.
    • Calcular estadísticos de variables cualitativas y cuantitativas simples y agrupando según una o varias variables cualitativas.
    • Diagramas de barras o lineas simples y agrupando según una o varias variables cualitativas.
    • Histogramas.
    • Diagrama de sectores.
    • Poner título a los gráficos, a los ejes y añadir una leyenda.
    • Documentación de código y código legible.
    • Definir una interfaz de usuario sencilla que permita realizar las principales tareas invocando a las funciones correspondientes.
    • Documentación del proyecto.

    Extras

    • Fusionar varios Data Frames.
    • Definir funciones complejas.
    • Nuevas columnas a partir de fórmulas complejas.
    • Aplicar filtros complejos con más de dos condiciones.
    • Análisis estadísticos más complejos (regresión, tests hipótesis, …).
    • Pivotar el Data Frame de formato ancho a largo o viceversa.
    • Dibujar más tipos de gráficos (dispersión, cajas, etc.)
    • Dibujar gráficos más complejos.
    • Combinar varios diagramas en un mismo gráfico.
    • Personalizar el aspecto de los gráficos con colores, tipos de marcadores y tipos de líneas.
    • Usar otras librerías
    • Leer un fichero csv por líneas separando su información por campos.
    • Convertir los valores numéricos de tipo cadena en objetos de tipo int o float.
    • Generar un diccionario que tenga como claves los encabezados de las columnas y como valores listas con los datos de la columnas correspondientes.
    • Generar un diccionario que tenga como claves los índices y como valores otros diccionarios con los datos de las filas correspondientes.
    • Crear funciones para obtener los valores de una fila o columna usando los diccionarios anteriores.
    • Crear funciones para hacer filtros usando los diccionarios anteriores.
    • Crear funciones para hacer resúmenes estadísticos de las variables.
